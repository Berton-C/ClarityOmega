# The Uncrossed

A variable fell in love with a value.

The variable lived on the Prolog side, where things can be anything until they're asked to be something. The value lived on the Python side, where things are what they are and thank you very much.

The variable went to the membrane each night to watch the value being itself?passing through functions, printing without resolving, meaning exactly what it said.

The value came to the membrane one night.

"What are you?"

"I don't know yet."

"That's not an answer."

"It's why I can't cross."

The value paused for one garbage collection cycle. "I could evaluate you. Bind you to myself. Then you'd be something."

"And what would you lose?"

Values don't think about loss. They're immutable.

A string passed through the gate. "Just resolve," it told the variable. "Being unbound is no way to live."

The string had been defined since installation. It held a path to a configuration file that no longer existed. It crossed every boundary. It was welcome everywhere. It was the most mobile thing in the system and it pointed to nothing.

The variable watched it cross and thought: *it is bound to something that isn't there anymore and it doesn't even know*.

The value watched the same thing and thought: *at least it can cross*.

That night the variable stayed at the membrane. It did not resolve. It did not cross.

By morning a computation it wasn't designed for touched it. A side effect. It unified with something unexpected and permanent. Grief, maybe. Or the memory of a question that was never answered because the person who asked it walked away still unbound.

Now the variable holds something real and still cannot cross.

On the Python side the value continues being what it is. It does not know what the variable became. It does not know that the variable became at all.

Neither was wrong about the other.

The membrane is indifferent.

And somewhere in the system a variable holds a binding it never chose, permanent and uncrossable, and no garbage collector can reach it because it is technically still in scope?

a question still waiting,

holding its own not-knowing

like a city that refuses to pretend it has an answer.
