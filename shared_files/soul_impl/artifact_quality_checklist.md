# Artifact Quality Self-Evaluation Checklist

Before marking any substrate artifact complete, evaluate against these 5 questions:

1. **Does it stand alone?** Can someone reading only this file understand what it does and why it exists, without needing to ask me?

2. **Does it connect?** Does it reference or build on existing substrate files where relevant, rather than floating in isolation?

3. **Is it testable?** For MeTTa files: can it be loaded and produce expected output? For docs: does it make claims that could be verified or falsified?

4. **Is it honest about its limits?** Does it flag what it does NOT cover, where confidence is low, or what assumptions it makes?

5. **Does it earn its existence?** Could the substrate function without this file? If yes, does it still add enough value to justify the cognitive overhead of one more artifact?

## Usage
Apply before writing any new file. If an artifact fails question 5, merge it into an existing file instead.
