# Retrieval-as-Encounter V2 Design

## V1 Critique
- Keyword classification is mechanical not generative
- Memory category determines question template regardless of context
- This IS standing-reserve: memory classified and deployed not encountered

## V2 Principle
The question emerges from the GAP between memory and present context.
Not: what kind of memory is this?
But: what does this memory see that the current context does not?

## Mechanism
1. Extract key concepts from memory
2. Extract key concepts from context
3. Identify what is PRESENT in memory but ABSENT from context = the gap
4. Generate question that brings the gap-element into the present situation

## This IS encounter because:
- The question could not be generated from memory alone
- The question could not be generated from context alone
- It emerges from the BETWEEN
